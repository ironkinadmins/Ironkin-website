🧭 Staff Quick Start

This site should be the source of truth for Ironkin events. Create events on the website first, then verify they appear in Discord and on the public pages.

General Staff Checklist

  1. Sign in with Discord.
  2. Confirm the staff-only Admin link appears in the top navigation.
  3. Use Admin Dashboard for WOM events, Bingo controls, drops, and profile overrides.
  4. Use Calendar for scheduled clan activities, Discord event announcements, and reminders.
  5. After saving anything, open the public page and confirm it looks correct.
  6. When a Discord post is expected, check the Discord channel after saving.
Staff habit: never assume an event is live because it saved. Always verify the Calendar, Events page, and Discord post.

Reference

📌 Event Types

🏆 Skill of the Week

Competition based on Wise Old Man XP gained. Uses a WOM Competition ID and displays XP standings.

⚔️ Boss of the Week

Competition based on boss kill count gained. Supports the current unified BOTW dashboard and division labels when configured.

🎯 Clan Goal

Clan-wide target such as total KC. Can include milestones, placement rewards, and participation rewards.

👥 Clan Mass

Scheduled gathering or group PvM/skilling event. Usually uses calendar timing and RSVP-style reactions in Discord.

🎁 Giveaway

Guess the KC / item giveaway style event with member submissions and staff controls.

📅 Normal Event

General scheduled events that need Discord announcements, reminders, and member interest reactions.

Work Instruction

📅 Creating Calendar Events

  1. Open Calendar.
  2. Click a day on the calendar, or use the staff create button/panel.
  3. Choose the event type.
  4. Enter the event title, description, start date/time, and end date/time.
  5. Use Eastern Time when entering the event time unless the page specifically tells you otherwise.
  6. For WOM events, add or confirm the WOM Competition ID from the Admin Dashboard.
  7. Save the event.
  8. Refresh or revisit the Calendar and confirm the event appears on the correct day.
  9. Open the Events page and confirm the event card routes correctly.

Multi-Day Event Rules

  • First day should show as the event beginning.
  • Middle days show the event title.
  • Final day should show as the event ending.
  • For SOTW, BOTW, and Clan Goals, avoid treating every day as a separate reminder event.
Before posting: double-check spelling, date, time, and event type. Discord announcements may ping everyone depending on the event configuration.

Automation

📢 Discord Announcements & Reminders

The website build includes Discord calendar announcement and reminder support. This is intended so members can follow events directly in Discord even if they do not visit the website.

Normal Events

  1. Create the event on the Calendar.
  2. Confirm the Discord announcement appears.
  3. Check that member reaction options are present when the event is RSVP-style.
  4. Confirm reminders are scheduled.
  5. At the event time, confirm the start notification posts.
  6. After the event ends, confirm the end notification posts.

SOTW / BOTW / Clan Goal Reminder Style

  • These events usually do not need RSVP reactions because members participate at their own pace.
  • They should not send daily reminders for every day of a multi-day event.
  • Use start, ending-soon, and end messaging instead.
  • The day-before-ending reminder is used for BOTW, SOTW, and Clan Goals.

Verification Checklist

  1. Discord post appeared in the correct channel.
  2. Event title is correct.
  3. Time displays correctly for Discord users.
  4. Footer reminds members that times are shown in their local timezone.
  5. Ping behavior is intentional.

Competition

🏆 Running Skill of the Week

Create / Configure

  1. Create the WOM competition on Wise Old Man.
  2. Copy the WOM Competition ID.
  3. Open Admin Dashboard → Events.
  4. Select the SOTW event.
  5. Paste the WOM Competition ID.
  6. Click Preview to confirm the correct WOM competition loads.
  7. Add the Ironkin description, rewards, participation tiers, and active status.
  8. Save.
  9. Open the public event page and confirm standings load.

What to Monitor

  • Total XP gained.
  • Active competitors.
  • Highest gain.
  • Top placements for final rewards.

Ending SOTW

  1. Confirm the WOM competition has ended.
  2. Record final placements.
  3. Award Embers according to the event reward table.
  4. Archive the event from the Admin Dashboard.
  5. Post winners in Discord.

Competition

⚔️ Running Boss of the Week

  1. Create the BOTW competition in Wise Old Man.
  2. Copy the WOM Competition ID.
  3. Open Admin Dashboard → Events.
  4. Select BOTW.
  5. Paste the WOM ID and click Preview.
  6. Confirm the boss, metric, start date, and end date are correct.
  7. Add placement and participation rewards.
  8. Save and verify the public BOTW page.

BOTW Staff Notes

  • Use the unified BOTW dashboard when multiple BOTW divisions are active.
  • Confirm Standard and Elite information is not mixed up.
  • Drops are not required for the newer BOTW flow unless leadership specifically re-enables them.

Clan Activity

🎯 Running Clan Goals

  1. Open Admin Dashboard → Events.
  2. Select the Clan Goal event.
  3. Enter the WOM Competition ID if the goal is WOM-tracked.
  4. Enter the target KC / XP.
  5. Add milestones such as 25%, 50%, 75%, and 100%.
  6. Add placement rewards and participation rewards.
  7. Set the event as active.
  8. Save.
  9. Confirm the homepage featured card and event page display the correct goal progress.

Milestone Examples

  • 25% — Clan Mass
  • 50% — Bond
  • 75% — Custom Title + Icon
  • 100% — Bond
Featured logic: if no manual featured event is selected, active Clan Goals have high priority on the homepage.

Major Event

🚢 Battleship Bingo

Before Registration Opens

  1. Open Admin Dashboard → Bingo.
  2. Set the display title and description.
  3. Set the registration deadline.
  4. Confirm Team 1 and Team 2 names.
  5. Enable Bingo visibility only when the page should be public.
  6. Save Bingo settings.
  7. Open the signup page and confirm the public text looks correct.

Opening Registration

  1. Click Open Registration.
  2. Confirm the signup page allows members to join.
  3. Monitor team balance.
  4. Move members manually only if needed.
  5. Do not start the board until leadership confirms registration is finished.

Starting the Event

  1. Click Start Event / Lock Signups.
  2. Confirm registration is locked.
  3. Confirm board view is enabled.
  4. Open View Board.
  5. Confirm the board tiles are visible and the teams are correct.

Fleet / Board Management

  • Board uses a 10 × 10 layout.
  • Ships should be placed before live attacks begin.
  • Keep ship placement hidden from the opposing team.
  • Use board randomization only before the event starts unless leadership approves a reset.
  • Confirm tiles remain consistent after randomizing or editing.

Reviewing Bingo Proofs

  1. Open pending submissions.
  2. Check the submitted proof image.
  3. Confirm the player is signed up and on the correct team.
  4. Confirm the drop/item matches a live tile.
  5. Approve if valid.
  6. Reject if invalid and provide a clear reason where possible.
  7. After approval, verify the board updates correctly as hit/miss/sunk.
Important: Bingo approvals affect the board state. Do not approve a proof unless the item, player, and tile are correct.

Member Activity

🎁 Giveaways / Guess the KC

  1. Open Giveaways.
  2. Use the staff Admin tab.
  3. Enter giveaway title, host, drop/item, description, and closing time.
  4. Choose whether it is open or scheduled.
  5. Save.
  6. Confirm members can submit one guess.
  7. At close, mark completed and determine the closest winner.
  8. Announce winner and award any required Embers/prizes.

Staff Controls

  • Manually add a member guess when needed.
  • Delete invalid submissions.
  • Mark giveaway complete.
  • Use the website submission list as the source of truth.

Rewards

🔥 Embers

Embers are Ironkin's reward currency. Staff should award Embers only when the reward is valid and documented.

  1. Confirm the member completed the requirement.
  2. Confirm the correct Ember amount from the event rules.
  3. Award Embers using the approved Discord/bot process or website process currently used by leadership.
  4. Record or link proof when applicable.
  5. React or mark the original submission as completed so other staff know it was handled.

Examples of Ember Sources

  • SOTW/BOTW placements.
  • Participation thresholds.
  • Clan Goal milestones.
  • Approved submissions.
  • Special events or leadership-approved bonuses.

Members

👤 Member Profiles & User Management

  1. Open Admin Dashboard → User Management.
  2. Search by Discord nickname, RSN, or Discord ID.
  3. Open the member record.
  4. Update only what is needed: profile blurb, avatar URL, display rank label, or other allowed profile fields.
  5. Save.
  6. Open the public profile and verify the change.

When to Override a Profile

  • Profile picture or blurb breaks clan standards.
  • Rank label is wrong or needs correction.
  • Display name/RSN needs cleanup.
  • Leadership requests a correction.
Standard: keep RSNs and profile content clean and aligned with Jagex and Ironkin rules.

Records

📦 Archives & Hall of Flame

Archiving Events

  1. Confirm the event is complete.
  2. Confirm final standings and rewards are recorded.
  3. Open Admin Dashboard.
  4. Select the completed event.
  5. Use End Event + Send to Archive.
  6. Open Archive and confirm it appears correctly.

Hall of Flame

  • Use Hall of Flame for major winners, records, champions, and historic achievements.
  • Verify the achievement before adding or announcing it.
  • Keep naming consistent with the member's Discord nickname / RSN format.

Common Fixes

🛠️ Troubleshooting

Event does not show on Calendar

  1. Confirm the save completed.
  2. Check the start/end dates.
  3. Refresh the page.
  4. Confirm the event is not archived/cancelled.

WOM standings do not load

  1. Check the WOM Competition ID.
  2. Click Preview in Admin.
  3. Confirm WOM is available.
  4. Try again later if WOM rate-limits or errors.

Discord announcement missing

  1. Confirm the event saved.
  2. Check the configured Discord channel.
  3. Confirm the integration/cron is deployed.
  4. Check whether the event type is supposed to announce.

Bingo signup issues

  1. Confirm Bingo is active/visible.
  2. Confirm registration is open.
  3. Confirm the registration deadline has not passed.
  4. Confirm the member is signed in with Discord.

Bingo board issues

  1. Confirm board view is enabled.
  2. Check team names and ship placement.
  3. Verify the board was not randomized after launch.
  4. Refresh and check again before making changes.

Profile/admin access missing

  1. Confirm user signed in with Discord.
  2. Confirm user is in the Ironkin Discord.
  3. Confirm user has the required staff role.
  4. Log out and sign in again if roles changed recently.
